## פתרונות

### פתרון 1: התעלם מההודעה (מומלץ לפיתוח) ✅

**פשוט הקלד את הפרטים ידנית**. זה בסדר גמור בפיתוח!

**אין צורך לתקן כלום.**

---

### פתרון 2: הפעל HTTPS בפיתוח (אופציונלי)

אם ההודעה מפריעה לך, תוכל להפעיל HTTPS גם בפיתוח:

#### שלב 1: ערוך את vite.config.js

הסר את ה-comment מהשורות:
```javascript
export default defineConfig({
  plugins: [react()],
  server: {
    allowedHosts: true,
    https: true,  // <-- הסר //
    port: 5173,
  },
  ...
})
```

#### שלב 2: הפעל מחדש את Vite

```bash
# עצור את הסרוור (Ctrl+C)
cd Frontend
npm run dev
```

#### שלב 3: פתח בדפדפן

```
https://localhost:5173  (שים לב ל-HTTPS!)
```

**אזהרה**: הדפדפן יראה אזהרת אבטחה על "Self-signed certificate" - זה תקין בפיתוח.

לחץ "Advanced" → "Proceed to localhost" (או דומה בהתאם לדפדפן).

---

### פתרון 3: השתמש ב-mkcert (מתקדם)

`mkcert` יוצר SSL certificate מקומי מהימן:

```bash
# התקן mkcert
npm install -g mkcert

# צור certificates
mkcert -install
mkcert localhost 127.0.0.1 ::1

# עדכן vite.config.js
server: {
  https: {
    key: './localhost-key.pem',
    cert: './localhost.pem',
  },
}
```

## שאלות נפוצות

### ❓ האם הנתונים שלי מוצפנים?
**כן!** הנתונים נשלחים ל-Tranzila דרך HTTPS, גם אם האתר שלך על HTTP מקומי.

### ❓ האם זה מסוכן?
**לא בפיתוח.** זה רק על localhost - אף אחד חוץ ממך לא יכול לגשת.

**בייצור - כן!** חייב HTTPS בייצור.

### ❓ האם הלקוחות שלי יראו את זה?
**לא**, אם האתר שלך בייצור עם HTTPS (שזה חובה).

### ❓ איך אני מוודא ש-Tranzila מוצפן?
בדוק את ה-Network tab בדפדפן:
1. פתח DevTools (F12)
2. לך ל-Network
3. עשה תשלום
4. תראה קריאה ל-`https://api.tranzila.com` - שים לב ל-**HTTPS** ✅
